# Scriptorium — Расшифровка рукописных документов

Веб-приложение для автоматической расшифровки рукописных текстов с помощью нейронной сети Qwen2.5-VL.

## О проекте

Scriptorium создан в рамках проекта «Цифровая кафедра». Основная задача — упростить работу историков, архивистов и исследователей с рукописными документами: письмами, дневниками, архивными материалами.

Пользователь загружает изображение или PDF, нажимает «Распознать» — модель переводит рукопись в редактируемый текст. Результат можно поправить прямо в браузере и экспортировать в Word, PDF или TXT.

## Возможности

- Загрузка изображений (JPG, PNG, TIFF) и многостраничных PDF
- Распознавание текста через модель Qwen2.5-VL-3B (запускается в Google Colab на GPU)
- Редактор с цветовой разметкой источника каждого слова (OCR / база почерков / сокращения)
- Настройка порога уверенности, яркости и контраста документа
- Архив расшифрованных документов
- Экспорт в DOCX, PDF, TXT
- Пакетная обработка нескольких документов
- URL-маршрутизация: `/home`, `/archive`, `/ocr`, `/profile`

## Архитектура

```
браузер (статика на GitHub Pages)
    ↕ HTTPS
Google Colab (Flask + ngrok)
    ↕
Qwen2.5-VL-3B (GPU T4)
```

**Фронтенд** — чистый HTML/CSS/JS без фреймворков, хостится на GitHub Pages.

**Бэкенд** — Python-скрипт запускается в Google Colab, пробрасывается наружу через ngrok. Никаких затрат на сервер.

## Структура файлов

```
scriptorium/          ← фронтенд (GitHub Pages)
├── index.html

```

## Запуск

### 1. Бэкенд (Google Colab)

1. Открой `colab/scriptorium_colab.ipynb` в Google Colab
2. Включи GPU: **Среда выполнения → Сменить среду выполнения → T4 GPU**
3. Вставь ngrok-токен в ячейку 2 (бесплатно на [dashboard.ngrok.com](https://dashboard.ngrok.com))
4. Запускай ячейки по порядку (1 → 2 → 3 → 4)
5. После запуска ячейки 4 в консоли появится публичный URL

### 2. Фронтенд

Вставь URL из Colab в `js/api.js`, строка 6:

```js
var SPACE_URL = 'https://xxxx.ngrok-free.app';
```

Открой сайт на GitHub Pages — готово.

> URL действует пока активна Colab-сессия. При перезапуске ячейки 4 URL меняется — нужно обновить `api.js`.
