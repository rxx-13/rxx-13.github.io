# Спецификация требований к ПО (SRS)

**Проект:** Scriptorium — система автоматической транскрипции рукописных документов  
**Версия:** 1.2  
**Дата:** 2026-04-23

---

## 1. Глоссарий

| Термин | Определение |
|--------|-------------|
| **OCR** | Optical Character Recognition — преобразование изображения текста в машиночитаемый текст |
| **Pipeline (конвейер)** | Последовательность этапов обработки: OCR-распознавание → коррекция по базе почерков → расшифровка сокращений |
| **Qwen2.5-VL** | Мультимодальная нейросетевая модель (vision-language). Используется как OCR-движок |
| **База почерков (HW_DB)** | Словарь типичных ошибок распознавания рукописи (например, «тго» → «его») |
| **База сокращений (ABBR_DB)** | Словарь исторических и архивных сокращений русского языка (например, «губ.» → «губерния») |
| **Уверенность (confidence)** | Числовой показатель (0–100%) достоверности распознанного слова |
| **Порог уверенности** | Настраиваемый пользователем уровень: слова ниже порога визуально выделяются как сомнительные |
| **Viewport** | Область просмотра изображения в редакторе с поддержкой масштабирования, перемещения и поворота |
| **Архив** | Хранилище результатов распознавания в рамках текущей сессии браузера |
| **Пакетная обработка** | Режим последовательного автоматического распознавания нескольких документов |
| **HuggingFace Inference API** | Облачный API HuggingFace для запуска нейросетевых моделей без локального GPU; используется как бэкенд OCR-распознавания |
| **SPA** | Single Page Application — все экраны переключаются без перезагрузки страницы |

## 2. Описание системы

### 2.1 Назначение

Scriptorium — веб-приложение для автоматической транскрипции рукописных документов (писем, дневников, архивных материалов) с отсканированных изображений и PDF в редактируемый цифровой текст.

### 2.2 Решаемая проблема

Ручная расшифровка рукописных исторических документов — трудоёмкий процесс, требующий специальных навыков палеографии. Scriptorium автоматизирует первичную расшифровку с помощью нейросетевой модели и дополнительных словарей коррекции, снижая трудозатраты исследователей.

### 2.3 Целевая аудитория

- **Историки и исследователи** — работают с архивными рукописными источниками
- **Архивисты** — оцифровывают фонды рукописных документов
- **Студенты гуманитарных специальностей** — изучают первоисточники

### 2.4 Роли пользователей

В текущей версии системы существует одна роль — **Пользователь**. Система не требует регистрации: все операции выполняются без авторизации. Разграничение прав доступа не предусмотрено.

### 2.5 Границы системы

**Входит в систему:**
- Загрузка и просмотр документов (JPG, PNG, PDF, TIFF)
- OCR-распознавание рукописного текста через нейросетевую модель
- Постобработка: коррекция по базе почерков и расшифровка сокращений
- Редактор с viewport и цветовой разметкой результатов
- Экспорт в форматы DOCX, PDF, TXT
- Архив и пакетная обработка документов в рамках сессии

**Не входит в систему:**
- Серверная база данных (данные не сохраняются между сессиями)
- Аутентификация и разграничение прав пользователей
- Совместная работа над документами
- Офлайн-режим работы
- Распознавание печатного текста

## 4. Требования к дизайну и UI

### 4.1 Визуальный язык

- **Стиль:** минималистичный SaaS-инструмент.
- **Цветовая схема:** холодный серо-белый фон (`#f0f2f8`), акцентный цвет — фиолетовый (`#6b21e8`) для кнопок, иконок, интерактивных элементов
- **Иконки:** монохромные, единообразные по стилю

### 4.2 Навигация и структура

- Пользователь попадает на главную страницу → нажимает CTA → переходит в рабочий интерфейс без авторизации
- Главная страница и рабочий интерфейс выполнены в едином визуальном стиле: фон, цвета, типографика
- Боковая панель рабочего интерфейса: монохромные иконки, лаконичные подписи

### 4.3 Принципы интерфейса

- **PRN01** — Простота: пользователь получает результат не более чем за 3 действия
- **PRN02** — Прозрачность: прогресс распознавания отображается на каждом этапе pipeline
- **PRN03** — Минимализм: интерфейс содержит только актуальные функции без лишних элементов
- **PRN04** — Отказоустойчивость: при сбое сервера пользователь получает понятное сообщение об ошибке

## 5. Пользовательские истории

### Загрузка документов
- Как пользователь, я хочу загрузить скан (JPG, PNG, PDF, TIFF) через кнопку или drag-and-drop, чтобы начать расшифровку
- Как пользователь, я хочу видеть список загруженных документов с миниатюрами и статусами, чтобы ориентироваться в своих материалах

### Распознавание
- Как пользователь, я хочу запустить OCR одной страницы одной кнопкой, чтобы получить текст без лишних действий
- Как пользователь, я хочу распознать все страницы многостраничного PDF, чтобы получить полный текст документа
- Как пользователь, я хочу выделить фрагмент изображения и распознать только его, чтобы обработать конкретный участок рукописи
- Как пользователь, я хочу включать и отключать этапы коррекции, чтобы контролировать точность результата

### Работа с результатами
- Как пользователь, я хочу видеть распознанный текст с цветовой маркировкой по источнику, чтобы оценить качество распознавания
- Как пользователь, я хочу нажать на слово и увидеть его положение на изображении, чтобы сверить с оригиналом
- Как пользователь, я хочу настроить порог уверенности, чтобы выделить слова с низкой точностью
- Как пользователь, я хочу скопировать текст в буфер обмена, чтобы вставить его в другое приложение

### Экспорт
- Как пользователь, я хочу скачать результат как TXT, DOCX или PDF, чтобы продолжить работу в удобном формате

### Архив и история
- Как пользователь, я хочу сохранить распознанный документ в архив сессии, чтобы вернуться к нему позже
- Как пользователь, я хочу видеть историю экспортов и повторно скачивать файлы

### Пакетная обработка
- Как пользователь, я хочу запустить распознавание всех загруженных документов одной кнопкой
- Как пользователь, я хочу пропустить отдельные документы в очереди или отменить обработку

## 6. Функциональные требования

- **FR-001** — Система должна обеспечивать загрузку файлов форматов JPG, PNG, PDF и TIFF с локального устройства пользователя
- **FR-002** — Система должна обеспечивать загрузку файлов путём перетаскивания (drag-and-drop) из файлового менеджера операционной системы в область загрузки приложения
- **FR-003** — Система должна отображать каждую страницу многостраничного PDF-файла как отдельный элемент и обеспечивать переход между страницами без перезагрузки страницы браузера
- **FR-004** — Система должна предоставлять возможность запуска OCR-распознавания для отдельной страницы загруженного документа
- **FR-005** — Система должна предоставлять возможность последовательного запуска OCR-распознавания для всех страниц загруженного документа с отображением прогресса в формате «Страница X из N»
- **FR-006** — Система должна позволять пользователю выделить прямоугольную область на изображении страницы и запустить OCR-распознавание только для этой области
- **FR-007** — Система должна предоставлять инструменты масштабирования (zoom in/out), перемещения (pan) и поворота изображения страницы в области viewport
- **FR-008** — Система должна обеспечивать переход между страницами загруженного документа без полной перезагрузки страницы браузера
- **FR-009** — Система должна отображать каждое распознанное слово с цветовой маркировкой по источнику: исходный результат OCR, слово скорректированное HW_DB, слово расшифрованное ABBR_DB; в интерфейсе должна быть доступна легенда цветов
- **FR-010** — Система должна позволять пользователю задавать числовой порог уверенности от 0 до 100% и выделять в редакторе слова, уверенность которых ниже заданного порога, в режиме реального времени без повторного запуска OCR
- **FR-011** — Система должна обеспечивать экспорт распознанного текста в файл формата TXT в кодировке UTF-8
- **FR-012** — Система должна обеспечивать экспорт распознанного текста в файл формата DOCX с сохранением структуры абзацев
- **FR-013** — Система должна обеспечивать экспорт распознанного текста в файл формата PDF
- **FR-014** — Система должна позволять сохранять результат распознавания документа в архив текущей сессии браузера
- **FR-015** — Система должна предоставлять возможность просмотра, повторного открытия и удаления записей архива сессии
- **FR-016** — Система должна хранить историю экспортов текущей сессии и предоставлять возможность повторного скачивания ранее сформированных файлов
- **FR-017** — Система должна обеспечивать последовательный автоматический запуск OCR-распознавания для всех загруженных документов в режиме пакетной обработки с возможностью пропуска отдельных документов и остановки всей очереди
- **FR-018** — Навигация между разделами приложения должна осуществляться без полной перезагрузки страницы браузера (SPA)

## 7. Нефункциональные требования

### Производительность
- **PER01** — Интерфейс должен реагировать на действия пользователя (нажатие кнопок, переключение страниц) за время не более 200 мс
- **PER02** — Операции рендеринга страниц PDF и построения цветовой разметки результатов на стороне клиента должны выполняться за время не более 500 мс
- **PER03** — Время ожидания ответа от OCR-сервера не должно превышать 60 секунд для одной страницы документа

### Надёжность
- **DUR01** — При недоступности внешних сервисов (HuggingFace Inference API / Groq API) система должна отображать пользователю сообщение об ошибке с указанием причины и не прерывать работу с уже загруженными и распознанными документами
- **DUR02** — Система должна отображать сообщения об ошибках с указанием причины и рекомендуемого действия (например: «Сервер недоступен. Повторите запрос через несколько минут.»)

### Платформа и совместимость
- **PLT01** — Система должна корректно работать в актуальных версиях браузеров Chrome (v120+), Firefox (v121+) и Edge (v120+)
- **PLT02** — Интерфейс должен корректно отображаться на экранах с разрешением от 1280×768 пикселей
- **PLT03** — Все используемые браузерные API и сторонние библиотеки должны поддерживаться браузерами, указанными в PLT01

### Ограничения
- **CON01** — Данные архива и истории экспортов хранятся только в рамках текущей сессии браузера и не сохраняются между сессиями
- **CON02** — Максимальный размер одного загружаемого файла не должен превышать 50 МБ
- **CON03** — Функциональность OCR-распознавания недоступна при отсутствии подключения к внешнему OCR-сервису

### Дополнительные положения
- **EXT01** — В системе может быть реализована авторизация и личный кабинет пользователя с сохранением файлов между сессиями
- **EXT02** — Реализация этих функций не гарантируется и не является приоритетной на текущем этапе
