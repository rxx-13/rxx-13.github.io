# SRS — Scriptorium: Расшифровка рукописных документов

**Версия:** 1.1  
**Дата:** 2026-04-09  
**Статус:** Актуальная (после рефакторинга UI)

---

## 1. Глоссарий

| Термин | Определение |
|--------|-------------|
| **OCR** | Optical Character Recognition — преобразование изображения текста в машиночитаемый текст |
| **Pipeline (конвейер)** | Последовательность этапов обработки: OCR → коррекция по базе почерков → расшифровка сокращений |
| **Qwen2.5-VL** | Мультимодальная нейросетевая модель (vision-language). Используется как OCR-движок |
| **База почерков (HW_DB)** | Словарь типичных ошибок распознавания рукописи (например, «тго» → «его»). 8 записей, захардкожены в editor.js |
| **База сокращений (ABBR_DB)** | Словарь исторических архивных сокращений (например, «губ.» → «губерния»). 30 записей, захардкожены в editor.js |
| **Уверенность (confidence)** | Числовой показатель (0–100%) достоверности распознанного слова. Для Qwen всегда 80% — модель не возвращает пословную уверенность |
| **Порог уверенности** | Настраиваемый пользователем уровень: слова ниже порога визуально выделяются как сомнительные |
| **Viewport** | Область просмотра изображения в редакторе: масштабирование, перемещение, поворот |
| **Архив** | Хранилище результатов в оперативной памяти браузера. Данные теряются при перезагрузке страницы |
| **Пакетная обработка** | Режим последовательного автоматического распознавания всех загруженных документов |
| **Colab / ngrok** | Google Colab — облачная среда для запуска OCR-сервера на GPU. ngrok — туннель для публичного доступа к серверу |
| **SPA** | Single Page Application — все экраны приложения переключаются без перезагрузки страницы |

## 2. Описание системы

### 2.1 Назначение

Scriptorium — веб-приложение для автоматической расшифровки рукописных документов (писем, дневников, архивных материалов) с отсканированных изображений и PDF в редактируемый цифровой текст.

### 2.2 Целевая аудитория

Историки, архивисты, студенты гуманитарных специальностей — все, кто работает с рукописными историческими документами.

### 2.3 Текущий статус проекта

Scriptorium — прототип (MVP). Приоритет: работающий минимальный функционал, а не полнота функций. Всё нереализованное убрано из интерфейса.

### 2.4 Что система НЕ делает

- Не хранит данные между сессиями — при перезагрузке страницы всё теряется
- Не требует авторизации — страницы `/login` и `/register` удалены, пользователь попадает в приложение напрямую
- Не обеспечивает совместную работу нескольких пользователей
- Не работает офлайн — требуется запущенный OCR-сервер в Colab
- Не распознаёт печатный текст — ориентирован на рукописный

## 4. Требования к дизайну и UI

> Эти требования добавлены в версии 1.1 в рамках рефакторинга интерфейса.

### 4.1 Визуальный язык

- **Стиль:** минималистичный SaaS-инструмент. Ориентиры: Notion, Smallpdf, ILovePDF
- **Цветовая схема:** холодный серо-белый фон (`#f0f2f8`), один акцентный цвет — фиолетовый (`#6b21e8`) для кнопок, иконок, интерактивных элементов
- **Типографика:** Inter для UI, JetBrains Mono для распознанного текста
- **Иконки:** монохромные, единообразные. Emoji и цветные иконки папок — убрать

### 4.2 Структура и навигация

- Пользователь попадает на главную страницу → нажимает CTA («Начать») → попадает сразу в рабочий интерфейс без авторизации
- Главная страница и `/home` объединены в единый визуальный стиль: одинаковый фон, цвета, типографика
- Сайдбар `/home` — упрощённый: только текст или монохромные иконки, без цветных эмодзи

### 4.3 Принцип MVP

- На сайте присутствует только то, что реально работает
- Нереализованные элементы (селектор движка, выбор языка, кнопка «Сохранить проект») убраны из интерфейса
- Секция «Инструменты» на главной: только карточки с рабочим функционалом

## 5. Пользовательские истории

### Загрузка документов
- Загрузить скан (JPG, PNG, PDF, TIFF) через кнопку или drag-and-drop
- Видеть список документов с миниатюрами и статусами

### Распознавание
- Запустить OCR одной страницы одной кнопкой
- Распознать все страницы многостраничного PDF
- Выделить фрагмент изображения и распознать только его
- Включить/отключить коррекцию по базе почерков и расшифровку сокращений

### Работа с результатами
- Видеть распознанный текст с цветовой маркировкой по источнику (OCR / база почерков / сокращение / низкая уверенность)
- Нажать на слово — увидеть его положение на изображении
- Настроить порог уверенности ползунком
- Скопировать распознанный текст в буфер обмена

### Экспорт
- Скачать результат как TXT, DOCX (с цветовой маркировкой) или PDF (через диалог печати)

### Архив
- Сохранить распознанный документ в архив текущей сессии
- Просмотреть, скачать или удалить записи архива
- Просмотреть историю экспортов, повторно скачать файл

### Пакетная обработка
- Запустить распознавание всех загруженных документов одной кнопкой
- Пропустить отдельные документы в очереди
- Отменить пакетную обработку

## 8. Нефункциональные требования

- **Производительность:** клиентская обработка (HW_DB, ABBR_DB) — менее 300 мс. Время OCR зависит от GPU Colab, не нормировано
- **Безопасность:** данные передаются по HTTPS. API /predict не защищён авторизацией
- **Надёжность:** при ошибке одной страницы в пакетном режиме — продолжает обработку остальных. При потере связи — информативное сообщение
- **Совместимость:** любой современный браузер с поддержкой ES6, History API, Fetch API, Canvas API, Clipboard API
- **Ограничения:** сессия Colab ограничена по времени; при истечении — OCR недоступен
